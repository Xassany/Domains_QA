## 2. Категоризация функционала по степени важности

Вся функциональность разделена на три уровня приоритета исходя из целей абитуриента и бизнеса.

| Категория | Функциональная область | Обоснование приоритета |
| :--- | :--- | :--- |
| **Критический (Critical)** | **Смысловой контент** (Методология) | Если пользователь не поймет суть обучения (Peer-to-Peer), он не оставит заявку. Ошибки здесь недопустимы. |
| | **Целевые действия (CTA)** | Кнопка «Подать заявку» — это главная бизнес-цель страницы. Её неработоспособность блокирует весь процесс воронки. |
| **Важный (Important)** | **Адаптивность** | Около 50-70% трафика абитуриентов идет с мобильных устройств. Некорректная верстка приведет к потере аудитории. |
| | **Орфография и пунктуация** | Опечатки на сайте ИТ-школы подрывают доверие к качеству образовательного продукта. |
| | **Скорость загрузки (LCP)** | Медленная загрузка графики методологии увеличивает показатель отказов (Bounce Rate). |
| **Вспомогательный (Supporting)** | **Навигация (Футер)** | Второстепенные ссылки важны, но их временная неработоспособность не мешает основному процессу ознакомления. |
| | **Брендинг (Favicon)** | Наличие иконки влияет на узнаваемость вкладки, но не на функциональность обучения. |

---

## 3. Список проверок с идентификаторами (на базе Task 4)

Для матрицы использованы следующие ID проверок:

* **TC-METH-01:** Наличие блоков Peer-to-Peer, геймификация, отсутствие учителей.
* **TC-METH-02:** Работа вкладок/карточек в описании этапов.
* **TC-METH-03:** Корректная загрузка иконок и инфографики.
* **TC-METH-04:** Работоспособность кнопки «Подать заявку».
* **TC-METH-05:** Корректное отображение на мобильных устройствах.
* **TC-METH-06:** Работоспособность внутренних ссылок.
* **TC-METH-07:** Отсутствие «битых» ссылок (404).
* **TC-METH-08:** Проверка орфографии и пунктуации.
* **TC-METH-09:** Наличие Favicon и корректного Title.
* **TC-METH-10:** Контрастность текста на темных/фоновых блоках.
* **TC-METH-11:** Скорость загрузки тяжелого медиаконтента.
