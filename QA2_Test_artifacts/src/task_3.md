# Тест-план

**Проект:** Сайт «Школа 21»<br>
**Раздел:** Корпоративное обучение (Групповые интенсивы для руководителей и команд)<br>
**Дата составления:** 27.02.2026

---

### 1. Введение (Идентификатор документа)
Данный документ описывает стратегию, объем, ресурсы и график выполнения работ по тестированию раздела «Корпоративное обучение» на сайте «Школы 21». Цель тестирования — убедиться, что страница работает корректно, выглядит согласно дизайну и позволяет пользователям успешно взаимодействовать с контентом и формами заявок.

### 2. Объект тестирования (Test Items)
Клиентская часть (Frontend) веб-страницы по адресу:
[21-school.ru](https://21-school.ru/corporate-training/gruppovye-intensivy-dlia-rukovoditelei-i-komand)

### 3. Функции, подлежащие тестированию (Features to be tested)
* **UI и верстка:** Корректное отображение текстов, шрифтов, изображений и блоков на странице.
* **Навигация:** Работа всех внутренних и внешних ссылок, кнопок призыва к действию (CTA).
* **Интерактивные элементы:** Работа спойлеров, выпадающих списков, слайдеров (если применимо).
* **Формы обратной связи:** Валидация полей ввода (имя, телефон, email), успешная отправка заявки, появление сообщений об ошибках при некорректном вводе.
* **Адаптивность (Responsive Design):** Корректное отображение страницы на различных разрешениях экранов (десктоп, планшет, мобильный телефон).

### 4. Функции, не подлежащие тестированию (Features not to be tested)
* Бэкенд-логика и базы данных (проверяется только отклик на клиенте).
* Производительность и нагрузочное тестирование серверов.
* Тестирование безопасности (уязвимости, SQL-инъекции, XSS-атаки).

### 5. Подход к тестированию (Approach)
Тестирование будет проводиться вручную (Manual Testing) методом «черного ящика» (Black-box testing) — без доступа к исходному коду страницы. Будут применяться техники тест-дизайна: классы эквивалентности и анализ граничных значений (для проверки полей ввода в формах).

### 6. Среда тестирования (Environmental Needs)
Тестирование будет проводиться в следующих условиях:
* **Десктопные браузеры:** Google Chrome (последняя версия), Safari, Яндекс.Браузер.
* **Мобильные устройства:** Эмуляция через DevTools браузера Chrome (разрешения для iPhone 12/13 PRO, стандартный Android-смартфон).
* **ОС:** Windows 10/11, macOS.

### 7. Критерии начала и окончания (Pass/Fail)
* **Критерии начала (Entry Criteria):** Наличие доступа к интернету, сайт доступен по указанному URL, подготовлен набор тест-кейсов (Test Suite) или чек-лист.
* **Критерии окончания (Exit Criteria):** Все запланированные тест-кейсы пройдены. Все найденные дефекты (баги) задокументированы. Нет блокирующих (Blocker) дефектов, мешающих работе с основным функционалом страницы.

### 8. Тестовые артефакты (Test Deliverables)
По итогам завершения процесса тестирования будут предоставлены следующие документы:
* Настоящий Тест-план.
* Чек-листы и/или тест-кейсы.
* Баг-репорты (отчеты о найденных дефектах).
